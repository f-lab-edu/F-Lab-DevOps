### 1-1. 프로젝트 폴더 구조
```markdown
url-shortener/
├── app/
│   ├── main.py              ← FastAPI 앱 시작점 (진입문)
│   ├── api/
│   │   └── routes/
│   │       ├── health.py    ← /healthz 엔드포인트
│   │       └── url.py       ← URL 단축 API
│   ├── core/
│   │   ├── config.py        ← 환경변수 로드 (.env 읽기)
│   │   └── database.py      ← DB 연결 & 세션 관리
│   └── models/
│       └── url.py           ← DB 테이블 정의 (ORM)
├── Dockerfile               ← STEP 3에서 작성
├── docker-compose.yml       ← STEP 5에서 작성
├── requirements.txt
├── .env                     ← 비밀 설정 (git 제외)
└── .gitignore
```
---
### 1-2. 각 파일의 역할
•	main.py     — FastAPI() 인스턴스 생성, 라우터 등록, 앱 시작점

•	routes/     — URL별 엔드포인트 함수 모음 (관심사 분리)

•	config.py   — DATABASE_URL 등 환경변수를 .env에서 안전하게 읽어옴

•	database.py — DB 연결 풀(엔진) 생성, 요청마다 세션 열고 닫음

•	models/     — DB 테이블을 Python 클래스로 표현 (ORM 방식)

---
### 1-3. Python & FastAPI 핵심 개념
•	Python    — 우리가 쓸 프로그래밍 언어. 문법이 영어에 가까워 읽기 쉬움

•	FastAPI   — Python으로 API 서버를 빠르게 만드는 라이브러리

•	uvicorn   — FastAPI 앱을 실제로 실행시키는 웹 서버 (엔진 역할)
